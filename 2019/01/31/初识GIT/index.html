初识 “GIT”


1、把master同步到你自己的本地yongxin分支。

2、提交本地yongxin分支到远程yongxin分支

3、本地分支合并到master，并调到远程master分支。

注意:

  • 严格的代码管理，不允许每一个人都去合并master，应该有一个专人来去合并master。
  • 还要明白git是分布式的代码仓库管理，每个人都有本地master分支和远端的master分支，自己的分支也是这样。
